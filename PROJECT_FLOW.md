## 🔐 Authentication Flow

```
┌─────────────────────────────────────────────────────────────┐
│                    USER VISITS LOGIN PAGE                    │
│                    (/login)                                  │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│              USER ENTERS EMAIL & PASSWORD                   │
│              Clicks "Login" Button                           │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│           Login Component Calls authContext.login()          │
│           api.login(email, password)                        │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│              API CALL: POST /api/auth/login                 │
│              { email, password }                             │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│              BACKEND VALIDATES CREDENTIALS                  │
│              Checks user status (active/inactive)           │
└────────────────────┬────────────────────────────────────────┘
                     │
        ┌────────────┴────────────┐
        │                         │
        ▼                         ▼
    VALID                  INVALID
    CREDENTIALS            CREDENTIALS
        │                         │
        ▼                         ▼
┌───────────────┐         ┌───────────────┐
│ RETURN TOKEN  │         │ RETURN ERROR  │
│ + USER DATA   │         │ (401/403)     │
└───────┬───────┘         └───────┬───────┘
        │                         │
        ▼                         ▼
┌─────────────────────────────────────────────────────────────┐
│        FRONTEND STORES TOKEN IN localStorage                │
│        localStorage.setItem('auth_token', token)            │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│        FETCH COMPLETE USER WITH PERMISSIONS                  │
│        api.getCurrentUser()                                   │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│        BACKEND RETURNS:                                      │
│        {                                                     │
│          id: 23,                                             │
│          name: "hunain",                                     │
│          email: "user@example.com",                          │
│          role: { id: 5, name: "DEVELOPER" },                 │
│          permissions: [                                      │
│            "dashboard.view",                                 │
│            "projects.view",                                  │
│            "projects.create",                                │
│            "tasks.view",                                     │
│            ...                                               │
│          ],                                                  │
│          profile_image: "url..."                             │
│        }                                                     │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│        STORE USER DATA IN localStorage                       │
│        localStorage.setItem('user', JSON.stringify(user))    │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│        UPDATE AUTHCONTEXT STATE                              │
│        setState({                                            │
│          user: fullUser,                                     │
│          token: token,                                       │
│          isAuthenticated: true,                             │
│          isLoading: false                                    │
│        })                                                    │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│        REDIRECT TO DASHBOARD                                 │
│        navigate('/dashboard')                                │
└─────────────────────────────────────────────────────────────┘
```

## 🔄 Summary of Key Flows

1. **Authentication Flow**: Login → Token Storage → User Data Fetch → Permission Check → Dashboard
2. **Authorization Flow**: Permission Check → Grant/Deny Access → Conditional Rendering
3. **Data Fetching Flow**: Component Mount → React Query → Service → API → Cache Update → Render
4. **Real-time Updates Flow**: Socket Connect → Listen Events → Backend Emit → Handler Execute → UI Update
5. **CRUD Operations Flow**: Form Submit → Validation → API Call → Success/Error → Cache Invalidate → UI Update
6. **Navigation Flow**: Route Change → Permission Check → Component Mount → Data Fetch → Render
7. **Error Handling Flow**: API Error → Interceptor → Error Type Check → User Notification → Recovery Action

## 🎯 Quick Reference

- **Tech Stack**: React, TypeScript, Vite, shadcn-ui, Tailwind CSS, Supabase, React Query, Socket.io
- **State Management**: React Context (Auth), React Query (Server State), useState (Local State)
- **Routing**: React Router v6
- **Authentication**: JWT Token-based with localStorage persistence
- **Authorization**: Role-based permission system with granular permissions
- **Real-time**: Socket.io for live updates
- **Forms**: react-hook-form with Zod validation
- **UI Components**: shadcn-ui (Radix UI primitives)
- **Styling**: Tailwind CSS with dark mode support
- **API Client**: Axios with interceptors
- **Database**: Supabase (PostgreSQL)
